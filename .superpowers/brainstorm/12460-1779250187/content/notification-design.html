1. 数据库设计 — Notification 表

Prisma Schema: Notification
model Notification { id Int @id @default(autoincrement()) userId Int? @map("user_id") // null=全员通知 type String @default("system") // repair/stock/customer/system/beauty title String // 标题 content String @db.Text // 内容 targetType String? @map("target_type") // 跳转类型: repair_order/sales_lead/part/customer targetId Int? @map("target_id") // 跳转目标ID isRead Boolean @default(false) @map("is_read") readAt DateTime? @map("read_at") createdAt DateTime @default(now()) @map("created_at") user User? @relation(fields: [userId], references: [id]) @@map("notification") @@index([userId, isRead]) }

3. 点击跳转逻辑

消息点击 → 跳转目标
消息类型 点击行为 targetType 跳转路由
🔧 新工单通知 打开工单详情抽屉 repair_order /repair/orders → 详情抽屉
📦 库存预警 跳转配件列表（搜索该配件） part /inventory/parts?keyword=刹车片
👤 新客户注册 打开客户详情抽屉 customer /customer/list → 详情抽屉
📋 状态变更 打开工单详情抽屉 repair_order /repair/orders → 详情抽屉
📢 系统公告 展开公告详情弹窗 - 弹窗展示全文
