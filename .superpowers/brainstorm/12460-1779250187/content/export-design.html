📊 导出功能 — 设计方案

报表导出 PDF · 列表导出 Excel · 后端生成文件流 · 前端触发下载

1. 技术方案对比

B

后端生成 + 前端下载（推荐）

Excel: exceljs 生成 .xlsx | PDF: jspdf 服务端渲染

优势

  • 不受前端DOM限制
  • PDF 中文无乱码
  • 大数据量不卡浏览器

实现

  • 新建导出专用 Controller
  • exceljs + @types/exceljs
A

纯前端生成

xlsx 库 + jspdf + html2canvas

优势

  • 不增加后端接口

劣势

  • PDF 中文需要额外字体
  • 大数据量页面卡顿

5. 后端 API 设计

统一导出接口
GET /export/excel?module=report&type=daily&startDate=...&endDate=... → 返回 .xlsx 文件流 (Content-Disposition: attachment) GET /export/pdf?module=report&type=daily&startDate=...&endDate=... → 返回 .pdf 文件流 GET /export/excel?module=customers&keyword=...&pageSize=all → 导出全部客户数据 参数说明： module = report | customers | parts | repair | sales type = daily | weekly | monthly | summary pageSize = all 表示导出全部（不传则按分页导出当前页）
